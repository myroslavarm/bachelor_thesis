\begin{abstract}
\addchaptertocentry{\abstractname} % Add the abstract to the table of contents
Code completion is one of the essential features of any IDE that greatly improves the developer experience and productivity. Good code completion can both speed up the development process, as well as aid the developer in API exploration. On the other hand, having a slow or inaccurate completion can be very cumbersome. Thus, it is important to find a way to make is as effective as possible.

The current implementation of code completion in the Pharo IDE is based on the abstract syntax tree (AST) of source code. This allows us to know the semantic role and the type of tokens. Hence, at every point in the completion, only contextually relevant results are suggested. However, currently there is no effective way to sort the completion results.

In this work I study how code completion can be improved by sorting completion results using statistical language models, such as the n-gram language models. With the help of the n-gram approach, I implement two sorting strategies (unigram- and bigram-based) where the most relevant results are shown first.

Finally, I use the quantitative and qualitative evaluation techniques to compare the performance of the n-gram based to the alphabetic sorter, which is a default sorter in the Pharo IDE. As a result of the evaluation, the unigram sorter had the best performance.

As the final result, both the sorting strategies are available for usage in the Pharo IDE as part of the \href{https://github.com/myroslavarm/CompletionSorting}{CompletionSorting} library, although only the unigram is fast enough to be comfortably used by actual developers. The enhancement of the bigram sorter is part of the future work of this project.
\end{abstract}
